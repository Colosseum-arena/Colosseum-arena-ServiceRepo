# develop 기반 Git 작업 규칙

이 저장소는 `develop` 브랜치를 기본 브랜치로 사용합니다.

## 흐름
1. `develop`에서 최신 변경 사항을 받습니다.
2. GitHub 이슈를 한국어로 생성합니다.
3. `develop`에서 이슈 번호 기반 브랜치를 생성합니다.
4. 작업 후 한국어 커밋 메시지로 커밋합니다.
5. `develop` 대상으로 PR을 생성합니다.
6. 리뷰 후 `develop`으로 머지합니다.

## 네이밍 규칙
- 이슈 제목: 한국어
- 브랜치 이름: `타입/이슈번호-작업-제목`
- 커밋 제목: `타입 (#이슈번호) : 작업 제목`
- PR 제목: 한국어

## 예시
- 이슈: `로드맵 수정 채팅 내역 조회 API 추가`
- 브랜치: `feat/264-로드맵-수정-채팅-내역-조회-api-추가`
- 커밋: `feat (#264) : 로드맵 수정 채팅 내역 조회 API 추가`
