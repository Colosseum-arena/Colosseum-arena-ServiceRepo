# 해커톤 실행 계획

## 목표
AI들이 서로 공격하고 토론한 뒤 최종 코드 하나로 합의하는 데모를 완성합니다.

이번 데모의 성공 기준은 완벽한 보안이나 완벽한 코드가 아닙니다. 심사위원이 "AI들이 협업하고, 충돌하고, 합의한다"는 흐름을 명확히 볼 수 있으면 성공입니다.

## 팀 역할 분담

### A: Core Logic / AI Engine
AI가 똑똑하게 보이도록 만드는 역할입니다.

주요 책임:
- 멀티 에이전트 프롬프트 설계
- 코드 후보 2~3개 생성
- Red Team 취약점 분석
- Blue Team 방어 및 패치 제안
- Judge 최종 선택 및 합의 로직 구현

핵심 흐름:

```text
input -> 코드 생성 -> 취약점 분석 -> 평가 -> 최종 선택
```

### B: CLI / Demo / UX
터미널에서 서비스가 살아 움직이는 것처럼 보이게 만드는 역할입니다.

주요 책임:
- CLI 인터페이스 구현
- 단계별 로그 출력
- AI 대화처럼 보이는 연출
- 결과 가독성 정리

핵심 역할:

```text
터미널 출력 = 제품
```

## 6시간 타임라인

### 0~1시간: 설계 및 기본 세팅
A:
- 프롬프트 구조 설계
- 에이전트 역할 정의

B:
- CLI 기본 틀 구축
- 실행 명령어와 출력 흐름 설계

### 1~3시간: 핵심 기능 구현
A:
- 코드 후보 2~3개 생성
- 취약점 분석 프롬프트 작성

B:
- 출력 포맷 구현
- 단계별 로그 구조 구현

### 3~5시간: AI 협업 및 토론 연출
A:
- Red Team, Blue Team, Judge 역할 분리
- 간단한 합의 로직 구현

B:
- 대화형 로그 출력 구현
- 공격, 방어, 판정 흐름이 끊기지 않도록 정리

예상 출력:

```text
Red: 이 코드는 SQL injection 가능성이 있습니다.
Blue: 파라미터 바인딩과 입력 검증을 추가하면 방어할 수 있습니다.
Judge: Universe B가 가장 안전하고 수정 비용도 낮습니다.
```

### 5~6시간: 데모 완성 및 Polish
- 로그 정리
- 에러 제거
- 실행 흐름 최적화
- 발표용 데모 스크립트 점검

## MVP 범위

### 반드시 포함할 기능
- 코드 후보 2~3개 생성
- 그럴듯한 취약점 분석
- AI 대화 로그 출력
- 최종 코드 1개 선택
- 한 번에 실행되는 CLI 흐름

### 제외할 기능
- 실제 보안 정확도 검증
- 복잡한 정적 분석
- 웹 UI
- 장기 실행 워크플로우

## 데모 흐름

### 1. 명령어 실행

```bash
multiverse-sec "login API 만들어줘"
```

### 2. 코드 후보 생성

```text
Universe A: OOP 스타일
Universe B: Functional 스타일
Universe C: Async 스타일
```

### 3. Red Team 공격

```text
Red:
- SQL injection 가능
- 인증 누락
- race condition 가능성
```

### 4. Blue Team 방어

```text
Blue:
- 파라미터 바인딩 필요
- 인증 미들웨어 추가 필요
- 상태 변경 구간에 동시성 제어 필요
```

### 5. Judge 토론 및 판정

```text
Judge:
- Universe B가 가장 안전합니다.
- 수정 범위가 작고 설명 가능성이 높습니다.
- 최종 코드로 Universe B를 선택합니다.
```

### 6. 최종 결과 출력

```text
Final Secure Code Generated
```

## 성공 포인트

### A가 집중할 것
- 취약점 이름을 구체적으로 보여주기
- 공격과 방어 설명을 자연스럽게 만들기
- 결과가 그럴듯하게 납득되도록 만들기

### B가 집중할 것
- 로그가 끊기지 않게 만들기
- AI들이 대화하는 것처럼 보이게 만들기
- 심사위원이 한눈에 흐름을 이해하도록 만들기

## 심사위원에게 보여줘야 할 인상
- AI끼리 서로 다른 의견을 낸다.
- 공격과 방어 과정이 보인다.
- Judge가 이유를 들어 최종 코드를 선택한다.
- 결과물이 데모로 한 번에 실행된다.

## 한 줄 전략
A는 뇌를 만들고, B는 쇼를 만든다.

## 최종 체크리스트
- [ ] 코드 후보 2~3개가 생성된다.
- [ ] 취약점 분석이 출력된다.
- [ ] Red, Blue, Judge 대화 로그가 출력된다.
- [ ] 최종 코드 하나가 선택된다.
- [ ] CLI 명령어 한 번으로 전체 흐름이 실행된다.

## 결론
이 프로젝트의 핵심은 정확도가 아니라 "AI 협업이 살아있는 것처럼 보이는 경험"입니다. 데모에서는 구현의 완성도보다 생성, 공격, 방어, 합의의 흐름이 끊기지 않는 것이 가장 중요합니다.
